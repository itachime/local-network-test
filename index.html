排查提示

常见失败原因
如果报 Failed to fetch，通常是服务未启动、端口不可达、证书不被信任、CORS 未放行，或浏览器拦截了本地网络访问。
如果页面运行在 HTTPS 下访问 HTTP 地址，部分浏览器可能按混合内容策略拦截；可以同时对比 HTTPS 域名和 HTTP 127.0.0.1 两个默认接口。
如果页面嵌入在 iframe 中，外层 iframe 需要允许本地网络访问，例如添加 allow="local-network-access"。
手机测试建议
127.0.0.1 指向当前手机本机，不是电脑。如果本地服务跑在电脑上，手机需要访问电脑的局域网 IP，或使用文档中的可信转发域名。
首次请求可能触发浏览器权限提示；允许后再点一次测试，观察状态码和响应体。
